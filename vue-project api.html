安装 element-ui

PC端使用比较流行、文档的 Element-UI。 官网： http://element.eleme.io/#/zh-CN

打开命令行(cmd)窗口，cd 到项目路径，即第一步初始化的项目路径,安装 Element-UI

 localhost:Desktop zhaorongchao$ cd my-project/
 localhost:my-project zhaorongchao$ npm install -S element-ui

命令解释：

npm：是安装node时自带安装好的包管理工具，项目中所有的第三方js库，ui库都通过 npm来安装

intall：是npm安装命令

-S：是 --save 命令的简写，意思是安装到 线上环境依赖，所有上线后用到的包都需要用这条命令安装。例如 jQuery、element-ui

-D：是 --save-dev 命令的简写，意思是安装到 开发环境依赖，所有上线后用到的包都需要用这条命令安装。例如 jQuery、element-ui

四、项目目录介绍 
 --myproject       (项目根目录)
   --build 	   (webpack 配置文件所在的目录，不常用)
   --config 	   (开发环境，线上环境，本地服务(webpack-dev-server) 相关配置文件所在的目录，不常用)
   --node_modules  (npm安装的所有文件都在这里，基本不会修改这个目录下的东西)
   --src           (开发文件所在目录，开发的内容都在这)
     --assets      (静态资源存放在这里，例如：图片。这里是“相对路径”静态资源存放的位置)
     --components  (自己开发的组件存放在这里，这个文件夹下可以根据开发需求新建文件夹，例如：mall(商城)，pay(支付))
     --router      (页面路由都在这里配置，控制页面跳转)
     --App.vue     (根组件，后续开发的所有组件都会渲染到这个组件里，可以理解为 html 中的 body)
     --main.js     (项目入口文件，所有需要全局使用的模块都需要在这个文件里引用；webpack打包时也是从这个文件开始
                   一层一层找内容的；重要文件,不经常编辑)
   --static        (静态资源存放的地方，例如：js，css。这里是绝对路径静态资源)
   --test          (单元测试存放在这里，基本没用，测试交给测试吧)
   --package.json  (使用 npm 安装的东西都会自动加入到这个文件中，方便查看已经安装的模块，版本等信息)

️️把新建的组件加到 路由 中

️️找到 src/router/index.js (这是路由配置文件)，把新增加的组件加入到路由中

import Vue from 'vue'
import Router from 'vue-router'
import HelloWorld from '@/components/HelloWorld'
import Test from '@/components/Test.vue' // 把新增的Test组件导入进来
import Next from '@/components/Next.vue' // 把新增的Next组件导入进来
                        
Vue.use(Router)
                        
export default new Router({
routes: [
        {
            path: '/',
            name: 'HelloWorld',
            component: HelloWorld
        },
        {
            path: '/test', //这里是路由到路径
            name: 'test',
            component: Test // 对应 import 名称，如 Test、Next
        },
        {
            path: '/next',
            name: 'next',
            component: Next // 对应 import 名称，如 Test、Next
        },
    ]
})

️️配置完成后，启动服务(npm start),然后在浏览器输入 localhost:8080/test 就可以看到新增加到组件了

七、安装 axios

axios 是 vue 官方推荐的 http 模块

安装：npm install -S axios

 localhost:my-project zhaorongchao$ npm install -S axios

在入口文件全局引用 axios，打开src/main.js, 

// The Vue build version to load with the `import` command
// (runtime-only or standalone) has been set in webpack.base.conf with an alias.
import Vue from 'vue'
import App from './App'
import router from './router'
import Element from 'element-ui' // 导入element-ui
import 'element-ui/lib/theme-chalk/index.css' //导入 element-ui 样式文件
import axios from 'axios' // 导入 axios

Vue.prototype.$axios = axios //把 axios 添加到全局
Vue.use(Element) //使用 vue 引用 element-ui
Vue.config.productionTip = false

/* eslint-disable no-new */
new Vue({
  el: '#app',
  router,
  components: { App },
  template: ''
})

配置完后就可以在所有组件中使用 axios 了

八、配置访问服务器地址

全局配置url是为了解决 http 访问路径集中配置，不需要修改每个 http 请求的 url

另外可以解决跨域问题

打开 config/index.js,找到 proxyTable,添加如下配置

const path = require('path')

module.exports = {
  dev: {

    // Paths
    assetsSubDirectory: 'static',
    assetsPublicPath: '/',
    proxyTable: {
      '/test/': {
        target: 'http://dev1-tactics.juzifenqi.com',
        changeOrigin: false,
        pathRewrite: {
          '^/test/': '/test/'
        }
      }
    },

这样就可以在组件中使用相对路径 test 代替 http://dev1-tactics.juzifenqi.com

如果服务器地址换了（例如 切换线上地址），可以直接修改配置文件
